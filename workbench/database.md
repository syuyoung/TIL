# Table (aka Relation)

* 데이터를 기록하는 곳

# Field (aka Column, Attribute)​

* 테이블의 열

# Record (aka Row, Tuple)​

* 테이블의 행

# Database (aka Schema)

* 테이블의 집합

# Primary Key(PK)

* 각 레코드의 고유한 값
* 관계형 데이터베이스에서 레코드의 식별자로 활용

# Foreign Key(FK)

* 테이블 필드 중 다른 테이블의 레코드를 식별할 수 있는 키
* 각 레코드에서 서로 다른 테이블 간의 관계를 만든 데 사용
